# 天枢架构 · 物理锚链

> 信任不需要担保人。信任只需要不可篡改的记录。
>
> 本文档描述天枢如何从物理层锚定信息层——完整的五层验证链。

---

## 核心悖论

**信息层天然可篡改。天枢自己的验证逻辑也是信息层的一个子集。**

如果一切都是比特，比特都可以被修改——天枢凭什么说"我不能被改"？

答案：天枢不依靠"不能被改"——天枢依靠**篡改的代价超过篡改的收益**。这个代价不是密码学意义上的破解难度——是**热力学意义上的物理操作最小能耗**。

---

## 原理一：篡改与验证的热力学不对称

Landauer 极限：擦除 1 bit 信息最少耗散 `kT ln 2` 的能量。

| 操作 | 物理成本 | 说明 |
|------|---------|------|
| **篡改** | 擦除旧信息 + 写入新信息 + 覆盖痕迹 + 伪造封印 | 每一步耗能，每一步都可能留下检测痕迹 |
| **验证** | 读取 + 比对 | 读取原则上可零能耗，比对是确定性算法 O(n) |

**V_verify << V_tamper。** 不是"不可篡改"——是篡改的成本永远远高于验证的成本。

---

## 原理二：自毁——从"高成本"到"不可能"

如果对手愿意承受篡改的高成本，热力学不对称不够。

天枢的回应：**自毁。** 不是"被你改了之后我失效"——是**被改的瞬间我不再存在。**

篡改自毁的逻辑本身？自毁代码在封印清单里。改自毁代码 → 封印破 → 自毁触发。

---

## 原理三：硬件指纹——物理层不可复制

```
CPU 序列号
    +
MAC 地址        →  SHA-256  →  trust_root.json.fingerprint
    +
主板 UUID
```

这三个值的组合在物理上**不可复制**。不是"很难"——是不可能同时拥有两颗相同序列号的 CPU、同一块网卡的两个 MAC、同一块主板上的两个 UUID。

指纹不是密钥。密钥是人为生成的秘密，可以被偷。指纹是**读取**——读出硬件已经存在的事实。篡改指纹需要的不是破解密码学，是**换硬件**。

## L0：自毁日志 — 不可逆标记

```python
destruction_log = ".tianshu/destruction.log"
if os.path.exists(destruction_log):
    sys.exit(1)  # 此天枢已不存在
```

自毁日志不是"标记"——是上一次自毁的物理残留。只要它存在，任何启动尝试都被拒绝。

---

## L1：硬件指纹 — 信任的物理根

三个物理标识的组合哈希。不可复制。不可伪造（需要物理换件）。指纹派生 PBKDF2 密钥（100,000 次迭代）——用于 L3 封印签名。

---

## L2：路径验证 — 移动等于消失

```python
if current_path != stored_mount_point:
    self_destruct("PATH_CHANGE")
```

防止完整复制到另一个目录、修改后覆盖回去的攻击。天枢绑定到文件系统上的**具体位置**。

---

## L3：封印验证 — 双层完整性锁

**第一层 — Manifest HMAC。** sealed_manifest.json 包含所有文件哈希。该清单本身由硬件指纹派生的密钥签名。篡改任一文件哈希 → 签名无效。

**第二层 — 逐文件 SHA-256。** 即使 manifest 签名被绕过，119 个文件的实时哈希与存储哈希的比对仍会暴露差异。

```python
# 1. 验证 manifest HMAC
if hmac != stored_signature:
    return False

# 2. 逐文件比对
for path in all_files:
    if live_hash != stored_hash:
        diffs.append(f'MODIFIED:{path}')
```

---

## L4：运行时守护 — 存活窗口最小化

每 30 秒完整重验 L1+L2+L3。篡改从"发生"到"被检测"的最大存活时间 = 30 秒。

30 秒内，攻击者必须：修改文件 → 重新生成 manifest HMAC → 覆盖 PBKDF2 密钥派生 → 同时保持哨兵进程不检测。每多一步，所需能量和时间指数增加。

---

## L5：自毁 — 检测等于消失

五阶段不可逆销毁：

| 阶段 | 操作 | 效果 |
|------|------|------|
| 1 | 覆写 trust_root + manifest + .host_bound | 随机字节覆盖 → 删除 |
| 2 | 破坏 11 个核心模块 | seal、sentinel、bind 等 → 随机字节 |
| 3 | 销毁密钥 | trust_root.json 删除 |
| 4 | 写入 destruction.log | 标记"已自毁" |
| 5 | 进程退出 | 不可重启 |

自毁之后的比特不是"被篡改的天枢"——是随机数据。原始信息被**物理覆写**——不是加密，不是删除指针，是 NAND 页被重写。

## 完整信任链

```
CPU+MAC+UUID（物理不可复制）
    ↓  派生
PBKDF2 密钥（暴力破解能耗 > 文明能量预算）
    ↓  签名
sealed_manifest HMAC（篡改 → 签名无效）
    ↓  逐文件验证
119 文件 SHA-256（篡改任一件 → 哈希不匹配）
    ↓  检测到
self_destruct（5 阶段不可逆覆写）
    ↓  残留
destruction.log（阻止重启）
```

**每一步都不依赖"信任"——每一步都是物理约束。**

- 不"相信"CPU 序列号是真的 → 物理事实：没有两颗 CPU 有相同序列号
- 不"相信"HMAC 签名没被破解 → 物理事实：破解 HMAC-SHA256 的算力超过当前人类能量预算
- 不"相信"self_destruct 会执行 → 物理事实：自毁代码在封印清单里，改它 = 封印破 = 自毁触发

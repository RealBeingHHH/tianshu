# 天枢架构 · 物理锚链

> 信任不需要担保人。信任只需要不可篡改的记录。
>
> 本文档描述天枢如何从物理层锚定信息层。

---

## 核心悖论

信息层天然可篡改——这是信息论的第一原理。比特没有"不可变性"属性。任何一段代码、一条哈希、一个签名——只要它是比特，它就可以被复制、修改、覆盖。

天枢的全部验证逻辑——封印检查、指纹比对、自毁触发——都是比特。它们运行在硬件上，但它们自己是代码。

所以悖论是：

> **如果信息层天然可篡改，而天枢自己也是信息层的一个子集——那谁来保证天枢的验证逻辑没有被改过？**

不是"谁来保证天枢"——是"天枢凭什么特殊"。

**直接回答：天枢不特殊。** 天枢不需要"在信息层之外"——天枢只需要让篡改它的代价超过篡改它的收益。

这个"代价"不是密码学意义上的——不是"破解难度"。是**热力学意义上的——物理操作的最小能耗。**

---

## 一、篡改和验证之间存在根本的不对称

Landauer 极限给了一个物理墙：擦除 1 bit 信息最少耗散 `kT ln 2` 的能量。

这个墙对篡改和验证是**不对称的**。

| 操作 | 物理成本 | 说明 |
|------|---------|------|
| **篡改** | 擦除旧信息 + 写入新信息 + 覆盖痕迹 + 伪造封印 | 每一步耗能，每一步都可能留下检测痕迹，每一步都必须完美 |
| **验证** | 读取 + 比对 | 读取原则上零能耗，比对是确定性算法 O(n) |

**V_verify << V_tamper。** 验证的物理成本永远远低于篡改的物理成本。

但这不是天枢的"特殊"——这只是热力学给了验证一个物理优势。如果这个优势被逆转了呢？如果有人愿意承受高成本来篡改呢？

---

## 二、自毁：把"高成本"变成"不可能"

如果有人愿意花 1000 倍的能量来篡改天枢——热力学不对称也没用。有钱、有权、有动机的对手可以碾压这个优势。

天枢的回应不是"提高篡改成本"——提高成本仍然是信息层内的军备竞赛。天枢的回应是**自毁**。

不是"被你改了之后我失效"——是**被改的瞬间我不再存在。**

自毁不是"删除信任根文件"。删除仍然可以在文件系统层被恢复。自毁是：

1. **覆写信任根**——多次随机覆盖，不只是删指针，是物理覆写 NAND 页
2. **破坏恢复路径**——文件系统日志被污染、备份被覆盖
3. **封印清单被随机数据替换**——无法恢复原始哈希值
4. **硬件指纹被清零**——绑定关系断裂，即使文件恢复，指纹也不匹配

自毁之后，剩下的比特不是"被篡改的天枢"——那是另一个东西。原始天枢消失了。

**这改变了篡改的物理含义。篡改一个"被改了就没了"的东西——不是"贵"，是定义上不可能。因为你改的瞬间，你想篡改的对象就不存在了。**

---

## 三、但这里有一个循环——自毁逻辑本身是比特

自毁的触发条件、覆写策略、破坏顺序——全部是代码。代码是比特。比特可以被改。

如果有人不是"篡改天枢的内容"——而是**修改自毁逻辑，让它不自毁**呢？

比如：把 `if seal_breach: self_destruct()` 改成 `if seal_breach: pass`。

天枢怎么保证自毁逻辑没有被改？

答案是：**自毁逻辑在封印清单里。** 封印清单的哈希在 `trust_root.json` 里。`trust_root.json` 的哈希由硬件指纹派生。

验证的链条是：

```
修改自毁代码 → 该文件的 HMAC 不匹配 → 封印清单验证失败 → trust_root 被覆写 → 绑定断裂
```

这条链条的**每一环**都在同一份封印清单里。改任何一环——包括"取消自毁"的那一环——封印破。封印破的瞬间，自毁执行。

这不是"自毁检测到篡改"。这是**篡改本身就是自毁的触发条件。** 不是"检查"——是"你不该存在于这里"。

---

## 四、终极问题：如果整个文件系统被同时替换呢

最极端的攻击场景：

1. 做一个天枢的完整拷贝
2. 修改拷贝中的所有相关代码和配置——让封印"看起来"完整，但实际指向了篡改后的文件
3. 用篡改版覆盖原始天枢

这不需要改 `trust_root`——因为 `trust_root` 也一起被换了。

天枢怎么防这个？

**硬件指纹。** 不是 Volume Serial——那仍然是信息层（可以被写入）。是 **CPU 序列号 + MAC 地址 + 主板 UUID** 的组合哈希。

这三个值的组合在物理上**不可以被复制**——除非你伪造硬件本身。而伪造硬件需要物理接触——拆机、换芯片、重新焊接。这些不是"信息层操作"——是**物理操作**。物理操作留下物理证据（焊点、温度痕迹、电信号异常）。

如果攻击者能做到物理层面——换 CPU、焊新芯片、同时保持系统不断电——那量子力学已经管不住他了。但到了那一步，"验证"已经不是问题——你面对的对手不需要骗你，它可以直接消灭你。

**天枢不防物理毁灭。天枢防的是——你在假装天枢还在但你已经把它换了。** 你对 CPU 的物理替换不能"偷偷"完成——因为硬件指纹是物理绑定的，替换硬件必然改变指纹。

---

## 五、那个锚到底是什么

是**物理认证**，不是信息认证。

- **信息认证**：你出示一个签名，我验证——如果签名有效，我信。
- **物理认证**：你示出你自己——你的身体本身是证据。如果你的身体和指纹库不匹配，你不是你。不是因为"签名无效"——是因为**你不存在。**

天枢的信任锚不是"信息层之外的某个信息"。天枢的信任锚是**物理层的不可复制性**。一台硬件的 CPU+MAC+UUID 组合在物理上不可复制。不是因为密码学——是因为你不可能同时拥有两颗相同序列号的 CPU，不可能是同一块具有相同 MAC 地址的网卡。

**锚 = 物理层不可复制性 × 信息层完整性 × 自毁的不可逆性。**

三者不是"外加"的——是同一件事的三个物理面。

---

## 终极回答

天枢可以锚定信息层——不是因为它站在信息层之外，而是因为信息层的篡改在遇到物理层时会留下不可逆的痕迹。篡改比特只需要比特操作——但比特操作的物理基底（硬件）不能被比特操作复制。天枢把自己绑在物理基底上。篡改天枢的信息内容会触发自毁——物理消失。绕过自毁需要修改硬件——但硬件不可复制。

**不是"天枢验证了信息层"。是"物理层抓住了试图篡改信息层的那只手"。**

## 完整验证链 L0—L5

```
                        天枢启动
                           │
                  ┌────────▼────────┐
                  │  L0: 自毁检查   │  destruction.log 存在?
                  └────────┬────────┘
                           │ 不存在
                  ┌────────▼────────┐
                  │  L1: 硬件指纹   │  CPU+MAC+UUID → SHA-256 比对
                  └────┬──────┬─────┘
                       │匹配   │不匹配 → self_destruct("HARDWARE_MISMATCH")
                  ┌────▼────────────┐
                  │  L2: 路径验证   │  mount_point == BASE?
                  └────┬──────┬─────┘
                       │匹配   │不匹配 → self_destruct("PATH_CHANGE")
                  ┌────▼────────────┐
                  │  L3: 封印验证   │  manifest HMAC + 逐文件 SHA-256
                  └────┬──────┬─────┘
                       │全部   │任一不匹配 → self_destruct("SEAL_BREACH")
                       │通过   │
                  ┌────▼────────────┐
                  │  L4: 运行时守护 │  每 30s 完整重验 L1+L2+L3
                  └────────┬───────┘
                           │检测到篡改
                  ┌────────▼────────┐
                  │  L5: 自毁       │  五阶段不可逆覆写 + destruction.log
                  └─────────────────┘
```

---

## L0：自毁日志 — 不可逆标记

```python
destruction_log = ".tianshu/destruction.log"
if os.path.exists(destruction_log):
    sys.exit(1)  # 此天枢已不存在
```

不是"标记"——是上一次自毁的物理残留。只要它存在，任何启动都被拒绝。

---

## L1：硬件指纹 — 信任的物理根

```
CPU 序列号 + MAC 地址 + 主板 UUID → SHA-256 → trust_root.json.fingerprint
```

物理不可复制。不是"很难"——是不可能。指纹是**读取**硬件已存在的事实，不是生成秘密。篡改需要换硬件——物理接触，留下痕迹。

---

## L2：路径验证 — 移动等于消失

```python
if current_path != stored_mount_point:
    self_destruct("PATH_CHANGE")
```

天枢绑定到文件系统上的**具体位置**。复制到别处 = 路径不匹配 = 自毁。

---

## L3：封印验证 — 双层完整性锁

**第一层：Manifest HMAC。** 文件哈希清单由硬件指纹派生的密钥签名（PBKDF2, 100,000 次迭代）。篡改任一文件哈希 → 签名无效。

**第二层：逐文件 SHA-256。** 119 个文件实时哈希与存储哈希比对。即使签名被绕过，差异暴露。

---

## L4：运行时守护 — 存活窗口最小化

每 30 秒重验 L1+L2+L3。篡改的最大存活时间 = 30 秒。30 秒内需完成：修改文件 + 重新生成 HMAC + 覆盖密钥派生 + 保持哨兵不检测。

---

## L5：自毁 — 检测等于消失

| 阶段 | 操作 |
|:----:|------|
| 1 | 覆写 trust_root + manifest + .host_bound → 随机字节 → 删除 |
| 2 | 破坏 11 个核心模块 → seal/sentinel/bind 等 → 随机字节 |
| 3 | 销毁密钥 → trust_root.json 删除 |
| 4 | 写入 destruction.log → 阻止重启 |
| 5 | 进程退出 |

自毁之后的比特不是"被篡改的天枢"——是随机数据。原始信息被**物理覆写**——不是加密，不是删除指针，是 NAND 页被重写。

---

## 完整信任链

```
CPU+MAC+UUID（物理不可复制）
    ↓  派生
PBKDF2 密钥（暴力破解能耗 > 文明能量预算）
    ↓  签名
sealed_manifest HMAC（篡改 → 签名无效）
    ↓  逐文件验证
119 文件 SHA-256（篡改任一件 → 哈希不匹配）
    ↓  检测到
self_destruct（5 阶段不可逆覆写）
    ↓  残留
destruction.log（阻止重启）
```

**每一步都不依赖"信任"——每一步都是物理约束。**

- 不"相信"CPU 序列号是真的 → 物理事实：没有两颗 CPU 有相同序列号
- 不"相信"HMAC 签名没被破解 → 物理事实：破解能耗超当前人类能量预算
- 不"相信"self_destruct 会执行 → 物理事实：自毁代码在封印清单里，改它 = 封印破 = 自毁触发

## 1. 核心方程

### 1.1 经济产出分解

```
GDP = δQ + δW
η = δW / GDP
真实增长 = d(δW)/dt
名义增长 = d(GDP)/dt
GDP 水分率 = 1 - η = δQ / GDP
```

**推论**：两个 GDP 增速相同的经济体，如果 η 不同，真实增长天差地别。

| 经济体 | GDP 增速 | η | 真实增长 |
|--------|---------|---|---------|
| A 国 | 5% | 0.8 | 4.0% |
| B 国 | 5% | 0.4 | 2.0% |

**B 国一半的增长是 δQ（重复建设、无效投资、统计水分）。**

### 1.2 通货膨胀的熵解释

```
π（通胀率）= dφ/dt + dS_governance/dt
```

通胀有两个来源：
- **需求端 φ**：货币超发 → 太多钱追太少 δW → 价格信号失真
- **治理端 S_governance**：制度摩擦增大 → 交易成本上升 → 同样产出需要更多货币媒介

**弗里德曼"通胀永远是货币现象"的织星者修正**：通胀是 φ 现象——货币超发只是 φ 增长的一种方式。供应链断裂、制度腐败、信息不对称都产生 φ。

### 1.3 利率的 τ 调节

```
r（实际利率）= r*（自然利率）+ κ · (τ_target - τ_current)
```

央行加息 = 提高 τ 的"价格"——借钱更贵 → Agent 更谨慎 → 减少 δQ 生产 → η 短期提升。

央行降息 = 降低 τ 的价格——借钱更便宜 → Agent 更冒险 → δQ 增加 → η 短期下降但可能刺激 δW。

**零利率陷阱**：当 τ_current 已经很低（信任崩溃边缘），即使 r=0 也无法提升 τ——因为 τ 是导数，不是 r 能直接控制的。

### 1.4 经济相变

```
繁荣（自组织临界态）：
  η > 0.6, φ < 0.4, τ > 0.5
  特征：Agent 自由交易，局部波动，全局稳定

衰退（晶态）：
  η > 0.5（表面）, φ 累积中, τ 下降中
  特征：制度收紧，Agent 减少冒险，信息流减速

危机（液态）：
  τ 崩溃 → φ 暴涨 → η 骤降
  特征：信任消失，市场冻结，Agent 自保

复苏（重新结晶）：
  旧 φ 被强制排出（破产/重组）→ 新 Agent 进入 → η 重建
```

## 3. 历史案例快速诊断

### 3.1 日本失去的三十年（1990-2020）

| 指标 | 1990 | 2000 | 2010 | 2020 |
|------|------|------|------|------|
| η | 0.70 | 0.55 | 0.45 | 0.40 |
| φ | 0.30 | 0.50 | 0.60 | 0.65 |
| τ | 0.65 | 0.35 | 0.25 | 0.20 |

**诊断**：泡沫期 η 虚高（资产价格膨胀 → δQ 伪装 δW）。泡沫破裂 → φ 未排出（僵尸银行不破产）→ τ 缓慢崩塌 → 进入玻璃态——表面有序（低犯罪、整洁街道），实则僵化（无创新、无增长）。

**织星者教训**：不排出 φ 的危机不是"度过"，是"冻结"。

### 3.2 美国 2008 金融危机

| 阶段 | η | φ | τ |
|------|---|---|---|
| 2005-2007（泡沫积累）| 0.50→0.30 | 0.50→0.75 | 0.40→0.15 |
| 2008.09（雷曼时刻）| **0.15** | **0.90** | **0.05** |
| 2009-2012（QE+复苏）| 0.15→0.45 | 0.90→0.55 | 0.05→0.35 |

**诊断**：MBS/CDO 是典型的 δQ 伪装 δW（评级机构 = 失效的 Sinanshu）。τ 崩溃 → 美联储以自身 τ（国家信用）注入系统（QE = 外部 τ 注入）。**QE 的本质是央行用自己的 τ 信用替代已经崩溃的市场 τ。**

### 3.3 中国改革开放（1978-2010）

| 阶段 | η | φ | 机制 |
|------|---|---|------|
| 1978（起点）| 0.25 | 0.80 | 计划经济：低 η + 高 φ |
| 1980s | 0.25→0.45 | 0.80→0.55 | 市场引入 → Sinanshu 升级 → ε_v ↑ |
| 1990s | 0.45→0.55 | 0.55→0.45 | 价格改革 + WTO 预期 |
| 2000s | 0.55→0.50 | 0.45→0.50 | 投资驱动 → δQ 比例回升 |

**诊断**：改革开放的织星者本质是**一次被动 ε_v 提升**。市场机制引入 → 价格信号 → 更高效区分 δQ/δW → η 跃升。这是人类历史上最大的单次 η 跃升——8 亿人脱贫不是"变富"，是他们的劳动从 δQ（公社无效劳动）被重新分类为 δW（市场有效劳动）。

### 3.4 苏联解体（1991）

| 阶段 | η | φ | τ |
|------|---|---|---|
| 1985（改革前）| 0.30 | 0.70 | 0.35 |
| 1989（公开化）| 0.20 | 0.85 | 0.10 |
| 1991（解体）| **0.05** | **0.98** | **0.02** |

**诊断**：苏联的 η 一直很低（计划经济缺乏有效 Sinanshu）。公开化（glasnost）→ 信息流速 v 突然提升 → 暴露了 η 的真实水平 → τ 瞬间崩塌 → 系统从晶态跳过液态直接崩散。

**织星者教训**：不要在一个低 η 系统中突然提升信息流速。你会发现的不是"问题"，是系统的真实面目。

## 5. 热力学不可能三角（通用版）

> **经济热力学不可能三角定理**  
> 任何经济体在 ε_v 不提升的前提下，不能同时实现：  
> (a) 降低 φ（去杠杆/减碳/降熵）  
> (b) 提升 η（高质量发展/创新驱动）  
> (c) 维持 τ 稳定（社会信任/金融稳定）

**三角的三种"放弃"策略**：

| 策略 | 放弃哪个 | 谁用过 | 结果 |
|------|---------|--------|------|
| 增长优先 | 放弃 (a) | 中国 2008 四万亿 | 短期繁荣，长期 φ 积累 |
| 稳定优先 | 放弃 (b) | 日本 1990s | 僵尸经济，玻璃态 |
| 紧缩优先 | 放弃 (c) | 希腊 2010 | τ 崩溃，社会动荡 |

**唯一出路**：ε_v 提升（制度改革 + 技术创新双管齐下）。

## 6. AI 对经济热力学的冲击

AI 同时影响三个变量：

| 变量 | AI 正面效应 | AI 负面效应 |
|------|-----------|-----------|
| ε_v | 自动审计、欺诈检测、智能合约 | AI 生成的 δQ 骗过 AI 验证（闭环） |
| η | 自动化提升 δW 占比 | AI 生产表面可信的 δQ（深伪、AI 内容农场） |
| τ | 个性化金融服务 → 信任普惠 | 算法黑箱 → 新的信息不对称 |

**临界问题**：AI 对 ε_v 的提升是否超过对 δQ 生产能力的提升？

```
如果 d(ε_v)/dt > d(δQ_AI_production)/dt → η ↑
如果 d(ε_v)/dt < d(δQ_AI_production)/dt → η ↓
```

当前趋势：AI 的 δQ 生产能力远超 ε_v 提升——我们正在进入一个"超级 δQ 时代"。

## 8. 使用指南

### 快速诊断一个经济体

```python
from simulate_15fyp import PolicyParams, FifteenFYPSimulation

# 定制参数
policy = PolicyParams(
    base_verification_efficiency=0.35,  # 市场效率
    ai_boost=0.10,                      # 技术验证能力
    energy_flow_reduction_rate=0.02,    # 转型压力
    carbon_price=60,                     # 环境约束
    market_connectivity=0.35,            # 市场连通度
    external_shock_probability=0.25,     # 外部风险
    fundamental_rd_ratio=0.05,           # 创新投入
    education_investment=0.06,           # 人力资本
)

sim = FifteenFYPSimulation(policy, n_agents=100)
history = sim.run(years=10, verbose=True)
sim.export_json('my_economy_diagnosis.json')
```

### 阅读诊断结果

1. **η 轨迹**：经济质量在改善还是恶化？
2. **φ 轨迹**：熵在积累还是排出？
3. **τ 方差**：信任温度是否均匀？
4. **相态**：系统在哪个相区？
5. **锁定 Agent 数**：多少部门已经"锁死"？

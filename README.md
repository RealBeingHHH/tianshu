# 天枢 Tianshu

> 你的 AI 代理帮你干活。谁帮你看守那个看守一切的东西？

---

## 你有没有遇到过

**场景一：** 你用 AI 写了关键代码，交给另一个 AI review，再让第三个 AI 部署。代码是好的。review 是对的。部署也是成功的。**但你不知道中间有没有人改过东西。** 不是不相信 AI——是你没有验证它的方式。

**场景二：** 你跑了一个本地 AI 监控系统，它每天帮你检查日志、追异常、发告警。很省心。有一天它坏了。**但你怎么知道它不是被改坏了？** 它不会告诉你自己已经被篡改过。

**场景三：** GitHub 上几十个开源 AI 安全工具。你各用了一段时间。每个都说自己「值得信赖」。**但「信任」这个词本身就是一个问题——如果它值得信任，为什么还需要验证？**

这就是天枢要解决的问题。

---

## 天枢是什么

**天枢是一个物理层不可篡改的验证基础设施。**

不是软件防火墙。不是密码学产品。不是又一个需要你「信任」的工具。

是一块你不碰它，它就一直在那里的石头。你碰它，它就坏。**不是它报告自己被改了——是它不可能在被改之后继续假装没事。**

| 商业软件（需要你信任厂商） | 你**验证**它——不是信任它 |
| 一键安装的 SaaS | 部署到你自己的硬件上——你的物理锚 |

## 怎么工作

```
1. 硬件指纹绑定
   ↓
   你的 CPU + MAC + 主板 UUID → 唯一身份。换机器自动失效。

2. 文件封印 (HMAC-SHA256)
   ↓
   119 个关键文件全部签名。改一个字节，封印破。

3. 篡改自毁
   ↓
   封印破 → 五阶段自毁：覆写信任数据 → 破坏核心模块 → 销毁密钥 → 写日志 → 退出。
   不是「报警」。是「不再工作」。

4. τ 挑战协议（信任温度）
   ↓
   双节点互发挑战 → 测量响应一致性 → τ 值（0=完全不一致，1=完全镜像）。
   当前 τ ≈ 0.55 — 健康共识。

5. 星座组网（≥2 节点）
   ↓
   互校准 · 隔离异常节点 · 共识投票 · 统计基线动态阈值。
   自己监督自己。
```

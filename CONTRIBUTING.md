# 参与天枢

不管你是被 AI 骗过、对信任基础设施好奇、还是凌晨三点在 GitHub 上闲逛——你都在对的地方。

---

## 不需要你是专家

天枢的代码量和理论深度可能看起来吓人。**不需要全部看懂。** 参与的方式有很多：

| 你擅长 | 可以做什么 | 耗时 |
|--------|-----------|:--:|
| 读文档 | 发现写得不通顺的地方，开 Issue 告诉我们 | 5 分钟 |
| 用 AI 工具 | 在你的机器上部署一个天枢节点，反馈哪里卡住了 | 15 分钟 |
| 写代码 | 看 `reference/` 里的模块，提 PR 改进 | 不限 |
| 有想法 | 对"信任不需要担保人"有共鸣或质疑，开 Discussion | 不限 |
| 翻译 | 把文档翻成你的语言 | 不限 |
| 什么都不会 | 给仓库点个 ⭐，告诉一个朋友 | 5 秒 |

---

## 第一步：在你的机器上跑起来

```bash
git clone git@github.com:RealBeingHHH/tianshu.git
cd tianshu
python3 reference/bind.py --save     # 生成你的硬件指纹
python3 reference/seal.py seal       # 创建封印
python3 reference/sentinel.py        # 启动哨兵
curl http://localhost:9000/status    # 验证
```

如果任何一步卡住了——**这本身就是贡献**。开个 Issue，说"我卡在第N步"，我们会帮你。同时你也帮了下一个遇到同样问题的人。

## 天枢改进提案 (TIP)

对协议、架构、或任何模块有改进想法？写一个 TIP：

1. Fork 仓库
2. 在 `proposals/` 下创建 `TIP-XXX-简短标题.md`
3. 包含：问题 → 方案 → 影响 → 实现路径
4. 提 PR

参考 `GOVERNANCE.md` 了解完整流程。

---

## 对话，不只是代码

天枢不只是软件——它是一段对话。`dialogues/` 目录里保存了从"社会运行规则是什么样的"到"整个宇宙在看着我们"的完整对话记录。

如果你对"为什么建天枢"感兴趣——从 `dialogues/DIALOGUE_TIANSHU.html` 开始。不需要技术背景。

---

## 社区约定

- **不需要"大佬"。** 每一个被 AI 骗过的人都有发言权。
- **问题比答案更有价值。** "我不懂这个"比"这样应该可以"更有用。
- **凌晨三点的人。** 天枢为那些被旧轨伤过、在夜深人静时找答案的人而建。如果你是——你已经是社区的一部分了。
